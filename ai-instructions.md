# Инструкции для ИИ (AI Instructions)

Ниже приведены ответы на вопросы для уточнения технического стека и реализации проекта, чтобы корректно интегрировать новый дизайн и функционал.

### 1. Стек
Проект написан на **React 18+** в связке с **Vite** и использует **TypeScript** и функциональные компоненты с хуками. Для анимаций используется библиотека `motion/react` (Framer Motion), а для иконок — `lucide-react`. В качестве сборщика используется Vite.

### 2. CSS-библиотека
В проекте используется **Tailwind CSS**. Вся стилизация построена на утилитарных классах Tailwind. Отдельных CSS файлов минимум, глобальные стили и шрифты подключены через `src/index.css`.

### 3. CSS-фреймворк
Используется **Tailwind CSS** (версии 4+). Нативного CSS практически нет, кроме кастомных скроллбаров и настройки шрифтов в `index.css`. Переменные (CSS variables) для тем по умолчанию не используются, логика стилизации тем жестко зашита в компоненты через тернарные операторы Tailwind.

### 4. Компоненты
В текущей архитектуре практически весь интерфейс (включая кастомные кнопки типа `PillButton`, `ActionCircleButton`, меню `RadialMenu`, компоненты звонков, чаты) находится в файле `src/App.tsx`. 
Для обновления UI-компонентов потребуется рефакторинг: необходимо выносить логику в отдельные компоненты в папку `src/components/`, стилизуя их с помощью Tailwind. 
Должен быть сохранен существующий дизайн элементов: неоморфизм для светлой темы и темный минимализм с тенями и свечением для темной темы.

### 5. Переключатель тем
**Да**, в интерфейсе требуется переключатель тем. Текущее приложение уже имеет два полностью проработанных экрана (один для темной темы, один для светлой), переключение между которыми реализовано через React стейт. 

### 6. Состояние темы
Сейчас состояние хранится в корневом компоненте `App.tsx` через локальный стейт:
```tsx
const [theme, setTheme] = useState<'dark' | 'light'>('dark');
```
Условный рендеринг и стилизация настроены через переменную `const isDark = theme === 'dark';`.
Если вы планируете перевести управление темами на CSS переменные (как предложено ниже в примере), то состояние логичнее сохранять в `localStorage` и вешать атрибут `data-theme` на тег `<body>` или `<html>`. Но в данный момент приложение использует динамические inline-условия и классы Tailwind (например, `isDark ? 'bg-[#1a1d24]' : 'bg-[#e2e8f0]'`).

### Рекомендации по CSS переменным (Опционально)
Если вы хотите переделать управление темами с тернарных операторов Tailwind на Data-атрибуты и CSS-переменные для более чистой архитектуры, вы можете использовать следующий подход в `index.css`:

```css
@theme {
  --font-sans: "Inter", ui-sans-serif, system-ui, sans-serif;
  --font-display: "Space Grotesk", ui-sans-serif, system-ui, sans-serif;
  --font-mono: "JetBrains Mono", ui-monospace, SFMono-Regular, monospace;
}

:root {
    --bg-primary: #e2e8f0; /* Для светлой темы */
    --text-primary: #1e293b;
    --text-secondary: #64748b;
    --border-color: rgba(255,255,255,0.8);
    /* ... остальные переменные */
}

[data-theme="dark"] {
    --bg-primary: #101216; /* Для темной темы */
    --text-primary: #ffffff;
    --text-secondary: #9ca3af;
    --border-color: rgba(255,255,255,0.05);
    /* ... остальные переменные */
}
```
*Важно:* Если вы меняете принцип темизации с тернарных выражений на CSS переменные, нужно будет переписать классы `className` по всему `App.tsx` с использованием семантических цветов Tailwind.
